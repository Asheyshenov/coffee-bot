# Analysis: Host Not Permitted Error

**Date:** 2026-03-12_02-52
**Feature:** feature/dev-callback-tunnel
**Issue:** Cloudflare tunnel requests rejected with "Host not permitted"

## Problem Statement

When accessing the callback server through a Cloudflare tunnel, Sinatra returns:
```
Host not permitted
```

This prevents external callbacks from reaching the callback server during development.

## Root Cause Analysis

### Location of Host Authorization

**File:** Sinatra4.2.1 default configuration
**Path:** `vendor/bundle/ruby/3.2.0/gems/sinatra-4.2.1/lib/sinatra/base.rb:1980`

```ruby
set :host_authorization, ->() do
  if development?
    {
      permitted_hosts: [
        "localhost",
        ".localhost",
        ".test",
        IPAddr.new("0.0.0.0/0"),
        IPAddr.new("::/0"),
      ]
    }
  else
    {}
  end
end
```

### How It Works

Sinatra4.x uses `Rack::Protection::HostAuthorization` middleware to prevent DNS rebinding attacks.

**File:** `rack-protection-3.2.3/lib/rack/protection/host_authorization.rb`

```ruby
# Blocks HTTP requests with an unrecognized hostname in any of the following
# HTTP headers: Host, X-Forwarded-Host, Forwarded
class HostAuthorization < Base
  default_options allow_if: nil,
                  message: 'Host not permitted'
end
```

### Why Cloudflare Tunnel Fails

When a request comes through Cloudflare tunnel:

1. **Request flow:**
   ```
   MWallet API → https://xxx.trycloudflare.com → Cloudflare Edge → localhost:4567
   ```

2. **Host header:**
   ```
   Host: xxx.trycloudflare.com
   ```

3. **Permitted hosts check:**
   ```ruby
   permitted_hosts: ["localhost", ".localhost", ".test"]
   # "xxx.trycloudflare.com" is NOT matched!
   ```

4. **Result:** Request is blocked with "Host not permitted"

### Current CallbackApp Configuration

**File:** [`lib/http/callback_app.rb`](lib/http/callback_app.rb)

```ruby
class CallbackApp < Sinatra::Base
  # Disable Sinatra logging (use our own)
  set :logging, false

  # JSON content type for all responses
  before do
    content_type :json
  end
  # ...
end
```

**Problem:** No `host_authorization` setting - uses Sinatra defaults.

## Solution Options

### Option 1: Disable Host Authorization (Development Only)

```ruby
# lib/http/callback_app.rb
class CallbackApp < Sinatra::Base
  # Disable host authorization in development for tunneling
  configure :development do
    set :host_authorization, { permitted_hosts: [] }
  end
  # ...
end
```

**Pros:**
- Simple to implement
- Works with any tunneling solution

**Cons:**
- Less secure in development
- Must ensure this never applies to production

### Option 2: Permit Tunnel Hosts via Environment Variable

```ruby
# lib/http/callback_app.rb
class CallbackApp < Sinatra::Base
  configure :development do
    # Permit additional hosts from environment (for tunneling)
    tunnel_hosts = ENV.fetch('TUNNEL_HOSTS', '').split(',').map(&:strip).reject(&:empty?)
    set :host_authorization, {
      permitted_hosts: ['localhost', '.localhost', '.test'] + tunnel_hosts
    }
  end
  # ...
end
```

**Usage:**
```bash
export TUNNEL_HOSTS=.trycloudflare.com,.ngrok.io
```

**Pros:**
- More secure - only specific domains allowed
- Flexible - works with any tunneling service
- Explicit configuration

**Cons:**
- Requires environment variable setup

### Option 3: Use allow_if Proc

```ruby
# lib/http/callback_app.rb
class CallbackApp < Sinatra::Base
  configure :development do
    set :host_authorization, {
      allow_if: ->(env) {
        # Allow all in development (use with caution)
        true
      }
    }
  end
  # ...
end
```

**Pros:**
- Maximum flexibility
- Custom logic possible

**Cons:**
- Completely bypasses protection
- Harder to audit

### Option 4: Bind to 0.0.0.0 (Not Recommended)

```ruby
set :bind, '0.0.0.0'
```

**This doesn't help** - the issue is Host header validation, not binding.

## Recommended Solution

**Option 2** is recommended for the following reasons:

1. **Security:** Only specific tunnel domains are permitted
2. **Flexibility:** Works with ngrok, cloudflare, localtunnel, etc.
3. **Explicit:** Configuration is visible in environment
4. **Production-safe:** Only applies in development environment

### Implementation

```ruby
# lib/http/callback_app.rb
class CallbackApp < Sinatra::Base
  # Disable Sinatra logging (use our own)
  set :logging, false

  # Configure host authorization per environment
  configure :development do
    # Permit tunnel hosts for development callbacks
    # Example: TUNNEL_HOSTS=.trycloudflare.com,.ngrok-free.app
    tunnel_hosts = ENV.fetch('TUNNEL_HOSTS', '.trycloudflare.com').split(',').map(&:strip).reject(&:empty?)
    set :host_authorization, {
      permitted_hosts: ['localhost', '.localhost', '.test'] + tunnel_hosts
    }
  end

  configure :production do
    # In production, only permit configured public host
    public_host = URI.parse(CoffeeBot::Config::PUBLIC_BASE_URL).host rescue nil
    set :host_authorization, {
      permitted_hosts: [public_host].compact
    }
  end

  # ... rest of the code
end
```

### Usage for Development

```bash
# .env.development
TUNNEL_HOSTS=.trycloudflare.com,.ngrok-free.app

# Start cloudflare tunnel
cloudflared tunnel --url http://localhost:4567

# The callback server will now accept requests from *.trycloudflare.com
```

## Security Considerations

### Development Environment
- Tunneling is inherently less secure than production
- Only accept callbacks from trusted payment providers
- Signature verification in `MWallet::Service.process_callback` provides additional protection

### Production Environment
- Host authorization should only permit the configured public host
- Never use tunnel hosts in production
- Ensure `PUBLIC_BASE_URL` is correctly configured
