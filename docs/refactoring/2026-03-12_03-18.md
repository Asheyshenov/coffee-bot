# Analysis: Polling Status Extraction Issue

**Date:** 2026-03-12_03:18
**Feature:** feature/fix-callback-runtime-config
**Issue:** `provider_status: nil` in StatusSyncer polling
**Order:** Order #31

## Problem Statement
The StatusSyncer logs show:
```
provider_status: nil, internal_status: :unknown
```

This means polling cannot extract the payment status from the MWallet API response.

## Diagnostic Results

### Raw API Response from MWallet
```json
{
  "version": 1005,
  "mktime": "177326389517983",
  "sid": "5058827638",
  "cmd": "statusPayment",
  "data": {
    "payments": [
      {
        "trans_id": "554205358044",
        "date_pay": "2026-03-12 03:13:35.179577",
        "amount": "40000",
        "status": "approved",
        "payment_id": "31",
        ...
      }
    ]
  }
}
```

### Current Code (Incorrect)
**File:** [`lib/services/mwallet/mwallet_service.rb:73`](lib/services/mwallet/mwallet_service.rb:73)
```ruby
provider_status = response.dig('data', 'status') || response.dig(:data, :status)
```

This returns `nil` because the response structure is:
```json
{
  "data": {
    "payments": [{ "status": "approved", ... }]
  }
}
```

The response has `data.status` (which doesn't exist), status is inside `data.payments[0].status`.

### Expected Code (Correct)
**File:** [`lib/services/mwallet/mwallet_service.rb:73`](lib/services/mwallet/mwallet_service.rb:73)
```ruby
provider_status = response.dig('data', 'payments', 0, 'status')
```

### StatusMapper Issue (Also incorrect)
**File:** [`lib/services/mwallet/status_mapper.rb`](lib/services/mwallet/status_mapper.rb)
```ruby
# MWallet status codes (integer)
PENDING   = 1
CANCELLED = 2
SUCCESS   = 3
...
end

```

The StatusMapper expects integer codes, but the API returns `"approved"` (string).

## Root Cause Analysis

### 1. Status Extraction Location
**File:** [`mwallet_service.rb:73`](lib/services/mwallet/mwallet_service.rb:73)
```ruby
response.dig('data', 'status')
```

- Response has `data.status` (doesn't exist)
- Response has `data.payments` (array)
- Code tries to access `payments[0].status` (but `payments` could be empty)

- Result: `nil`

### 2. Status Value Mapping
**File:** [`status_mapper.rb`](lib/services/mwallet/status_mapper.rb)
```ruby
# MWallet status codes (integer)
PENDING   = 1
CANCELLED = 2
SUCCESS   = 3
...
end
```

The StatusMapper expects integer codes, but the API returns `"approved"` (string).

## Recommendations

### Fix 1: Update Status Extraction in mwallet_service.rb
```ruby
# Extract status from payments array
def get_payment_status(order)
  response = client.get_status(order.invoice_id_provider)
  # Map status to internal format
  provider_status = response.dig('data', 'payments', 0, 'status')
  internal_status = StatusMapper.map_from_provider(provider_status)
  # ... rest of code
end
```

### Fix 2: Update StatusMapper to handle string status
