#### 2. Faraday Connection Created on Every Request
**Location:** [`mwallet_client.rb:68-75`](lib/services/mwallet/mwallet_client.rb:68), [`mwallet_client.rb:97-104`](lib/services/mwallet/mwallet_client.rb:97)

**Problem:** New connection object created for each API call. This prevents connection reuse and SSL session caching.

**Fix:**
```ruby
module MWallet
  class Client
    def initialize(config)
      # ... existing code ...
      @connection = build_connection
    end

    private

    def build_connection
      Faraday.new(url: @base_url) do |c|
        c.request :json
        c.response :json, content_type: /\bjson$/
        c.headers['Content-Type'] = 'application/json'
        c.options.timeout = @timeout
        c.options.open_timeout = @timeout
      end
    end

    def connection
      @connection
    end
  end
end
```

#### 3. DRY Violation: Signing Logic Duplication
**Location:** [`mwallet_client.rb:63-66`](lib/services/mwallet/mwallet_client.rb:63), [`mwallet_client.rb:93-95`](lib/services/mwallet/mwallet_client.rb:93)

The signing logic is duplicated in `create_invoice` and `get_status` methods, and also exists in `Signer` class but isn't used consistently.

**Before:**
```ruby
# In create_invoice
json_for_sign = JSON.generate(request_params)
hash = OpenSSL::HMAC.digest(OpenSSL::Digest.new('md5'), @password, json_for_sign).unpack1('H*')
request_params['hash'] = hash
```

**After:**
```ruby
# Use the Signer class consistently
def create_invoice(params)
  request_params = build_invoice_params(params)
  log_request('createInvoice', request_params)
  
  signed_params = @signer.build_signed_request(request_params)
  response = with_retry { connection.post('/api/json/json.php', signed_params) }
  
  handle_response(response, 'createInvoice')
end
```

#### 4. SRP Violation: Service Class Does Too Much
**Location:** [`mwallet_service.rb`](lib/services/mwallet/mwallet_service.rb)

The `Service` class handles:
- Invoice creation
- Status checking
- Status syncing
- Callback processing (should be separate)

**Refactoring:**
```ruby
module MWallet
  class Service
    # Invoice operations
    def create_invoice_for_order(order)
    end
    
    def get_payment_status(order)
    end
    
    def sync_status(order)
    end
  end

  # New class for callback handling
  class CallbackHandler
    def initialize(client)
      @client = client
    end
    
    def process(params, raw_body)
      # All callback logic here
    end
  end
end
```

#### 5. Pattern Matching Opportunity
**Location:** [`mwallet_service.rb:100-114`](lib/services/mwallet/mwallet_service.rb:100)

Ruby 2.7+ pattern matching could simplify status handling:

**Before:**
```ruby
case status_data[:internal_status]
when :paid
  if order.status == OrderStatus::INVOICE_CREATED
    order.mark_paid!(paid_amount: status_data[:paid_amount], paid_at: Time.now.utc)
    return true
  end
when :cancelled
  if [OrderStatus::INVOICE_CREATED, OrderStatus::PAID].include?(order.status)
    order.update(status: OrderStatus::CANCELLED)
    return true
  end
end
```

**After (Ruby 3.0+ Pattern Matching):**
```ruby
case { status: status_data[:internal_status], order_status: order.status }
in { status: :paid, order_status: OrderStatus::INVOICE_CREATED }
  order.mark_paid!(paid_amount: status_data[:paid_amount], paid_at: Time.now.utc)
  true
in { status: :cancelled, order_status: OrderStatus::INVOICE_CREATED | OrderStatus::PAID }
  order.update(status: OrderStatus::CANCELLED)
  true
else
  false
end
```

#### 8. Notifier: Potential Memory Leak with Tempfile
**Location:** [`notifier.rb:226-249`](lib/services/notifier.rb:226)

```ruby
def send_qr_image_from_url(chat_id, url)
  require 'tempfile'
  require 'open-uri'
  
  image_data = URI.open(url).read
  
  Tempfile.create(['qr', '.png']) do |file|
    # ... 
  end
end
```

**Issues:**
1. `require` statements inside method (should be at top)
2. No error handling for URL fetch failures
3. No timeout for HTTP request

**Fix:**
```ruby
require 'tempfile'
require 'open-uri'

def send_qr_image_from_url(chat_id, url)
  image_data = URI.open(url, open_timeout: 5, read_timeout: 10).read
  
  Tempfile.create(['qr', '.png']) do |file|
    file.binmode
    file.write(image_data)
    file.rewind
    
    bot.api.send_photo(
      chat_id: chat_id,
      photo: Faraday::UploadIO.new(file.path, 'image/png'),
      caption: "QR-код для оплаты"
    )
  end
rescue OpenURI::HTTPError => e
  log_error('Failed to fetch QR image', url: url, error: e.message)
  # Gracefully degrade - still send payment link without image
rescue StandardError => e
  log_error('Failed to send QR image from URL', chat_id: chat_id, url: url, error: e.message)
end
```
