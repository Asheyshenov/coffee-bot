**Overall Health Assessment: MEDIUM RISK**

The Coffee Bot project demonstrates a solid foundation with well-structured layered architecture, proper separation of concerns between models/services/handlers, and comprehensive payment integration. However, several architectural issues require attention:

- **SRP Violations:** The Router class handles too many responsibilities (737 lines)
- **DRY Violations:** Repeated keyboard building and message sending patterns

The codebase is production-viable with targeted refactoring of the identified areas.

### 2. Race Condition Fix (C2)

**Problem:** The claim_for_barista! method has a time-of-check to time-of-use (TOCTOU) race condition.

**Solution:** Use database-level advisory locks or optimistic locking with version column.

```ruby
# AFTER: lib/models/order.rb
def claim_for_barista!(barista_id)
  DB.transaction do
    # Use SELECT FOR UPDATE to lock the row
    locked_order = self.class.db[
      "SELECT id FROM orders WHERE id = ? AND status = ? AND assigned_to_barista_id IS NULL FOR UPDATE",
      id, OrderStatus::PAID
    ].first
    
    return false unless locked_order
    
    update(
      status: OrderStatus::IN_PROGRESS,
      assigned_to_barista_id: barista_id,
      assigned_at: Time.now.utc
    )
    
    log_info('Order claimed', order_id: id, barista_id: barista_id)
    true
  end
rescue Sequel::DatabaseError => e
  log_error('Order claim failed', order_id: id, error: e.message)
  false
end
```

### 3. Callback Transaction Safety (C3)

**Problem:** process_callback updates multiple tables without transaction protection.

```ruby
# BEFORE: lib/services/qrpay/qrpay_service.rb:286-336
def process_callback(params, raw_body, signature)
  callback_log = CallbackLog.log_callback(...)  # Write 1
  
  unless client.verify_callback_signature(...)
    callback_log.update(verified_signature: false)  # Write 2
    raise QRPayError::SignatureError.new('Invalid signature')
  end
  
  callback_log.mark_verified!  # Write 3
  
  if internal_status == :paid && order.status == OrderStatus::INVOICE_CREATED
    order.mark_paid!(...)  # Write 4 & 5 (order + payment)
  end
  
  callback_log.mark_processed!  # Write 6
end
```

**Solution:** Wrap in transaction with proper rollback.

```ruby
# AFTER: lib/services/qrpay/qrpay_service.rb
def process_callback(params, raw_body, signature)
  callback_log = CallbackLog.log_callback(...)
  
  unless client.verify_callback_signature(params, signature)
    callback_log.update(verified_signature: false)
    raise QRPayError::SignatureError.new('Invalid signature')
  end
  
  # Check for duplicate BEFORE processing
  if CallbackLog.duplicate?(params['eventId'], params['invoiceId'], raw_body)
    callback_log.update(verified_signature: true, processed: true)
    return true
  end
  
  DB.transaction do
    callback_log.mark_verified!
    
    order = find_order_for_callback(params)
    return true unless order  # Acknowledge but don't process
    
    process_payment_status(order, params)
    callback_log.mark_processed!
  end
  
  true
rescue Sequel::DatabaseError => e
  log_error('Callback processing failed', error: e.message)
  callback_log.mark_failed!(e.message)
  false  # Return false to signal retry
end
```

```ruby
module CallbackDispatch
  HANDLERS = {
    /\Acategory_(.+)\z/ => ->(c, m) { Handlers::Category.new(c).select(m[1]) },
    /\Aitem_(\d+)\z/ => ->(c, m) { Handlers::Item.new(c).select(m[1].to_i) },
    /\Aqty_(\d+)_(\d+)\z/ => ->(c, m) { Handlers::Cart.new(c).set_qty(m[1].to_i, m[2].to_i) },
    /\Acart_(add|checkout|clear|back)\z/ => ->(c, m) { Handlers::Cart.new(c).action(m[1]) },
    /\Aconfirm_(yes|no)\z/ => ->(c, m) { Handlers::Confirm.new(c).handle(m[1]) },
    /\Acheck_payment_(\d+)\z/ => ->(c, m) { Handlers::Payment.new(c).check(m[1].to_i) },
    /\Abarista_claim_(\d+)\z/ => ->(c, m) { Handlers::Barista.new(c).claim(m[1].to_i) },
    /\Abarista_complete_(\d+)\z/ => ->(c, m) { Handlers::Barista.new(c).complete(m[1].to_i) },
    /\Abarista_(queue|in_progress|menu)\z/ => ->(c, m) { Handlers::Barista.new(c).action(m[1]) }
  }.freeze
  
  def self.dispatch(callback, data)
    HANDLERS.each do |pattern, handler|
      if match = data.match(pattern)
        return handler.call(callback, match)
      end
    end
    log_debug('Unknown callback', data: data)
  end
end
```

```ruby
def self.expire_old_invoices(expire_minutes = nil)
  expire_minutes ||= CoffeeBot::Config::ORDER_EXPIRE_MINUTES
  cutoff_time = Time.now.utc - (expire_minutes * 60)
  
  # Single atomic UPDATE
  updated_count = Order.where(
    status: OrderStatus::INVOICE_CREATED,
    Sequel.lit('created_at < ?', cutoff_time)
  ).update(
    status: OrderStatus::EXPIRED,
    updated_at: Time.now.utc
  )
  
  log_info('Expired invoices', count: updated_count)
```
